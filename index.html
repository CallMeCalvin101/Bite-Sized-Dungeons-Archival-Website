Bite Sized Dungeons

    Project Requirements/Constraints
  • Online Multiplayer: game must have some sort of server component
  • Localization: game must support two different languages
  • Limited DexterityMust be played by only using 1 hand
  • Limited Energy & Attention: Natural 'resting' point of two minutes of active play
  • Limited Audition: Game must be completed by a player who cannot hear nay sounds from the game

Bite Sized Dungeons is a real-time online multipayer dungeon crawler and our project for CMPM 171: Game Design Studio.

The development for this project was split into three sprints consisting of three weeks each. The frontend for our game was developed using Phaser 3 and written in TypeScript. Our server uses Node.js and Socket.io for real-time client-server communication.

Below is a brief summary of what each group member did throughout the pre-production of the game

  • Vincent: Created core gameplay loop of the game, designed the classes, abilities, and enemies shown in the verticle slice, working on bringing the client and server to life
  • Sooin: Helped prototype out different libraries to facilitate the multiplayer of our game, currently looking into
  • Louis: Implemented the visuals and sounds of the game, including the background music, sound effects, and dragon icon of our game
  • Christian: Tested a working client and server prototype, implemented the splash screens, and currently working the client and server of our game
  • Alex: Helped define and implement our plans for the server and how to implement our game with a client-server model

Sprint 1: Research

During this sprint, we reasearched how we were going to build our game. This included looking into various tools that we should use during development.

8 low fidelity prototypes were created during this sprint, 2 by each team member.

An example of a prototype includes simple client-server communication

Sprint 2: Development

For this sprint we produced 8 high fidelity prototypes that we could build off of for the verical slice.

The focus for this was to create a compelling core gameplay loop as well as connecting our game server to our Phaser client.

During this phase, we incorporated GitHub Actions for continuous integration and continuous deployment (CI/CD) into our project.

Sprint 3: Vertical Slice

The final stretch of our project consisted of using everything we learned from sprints 1 & 2 and creating a vertical slice. You can view a video demoing our game here.
